Generative recommendation serving

SID-GR Inference

面向“长 context + 短 decode + 大 beam width”的推荐、搜索、广告生成式推理框架。 目标不是复刻通用 LLM serving，而是把 GR workload 的共享 context 和 beam 结构直接暴露给 runtime 与 kernel。

问题

长 context

用户、候选、历史行为等上下文可到 1k-5k tokens。

短 decode

真正生成通常只有几步，但每步需要保留大量候选。

大 beam

beam width 可到 128/256。通用框架会把它展开成大量普通 decode rows。

方案

GR-native runtime
  • ContextKV 按 request 共享。
  • BeamKV 只保存短 decode history。
  • BeamPath 显式记录 beam 转移。
  • continuous batching 直接服务 GR 请求形态。
专用 hot path
  • real gr-decode_atten decode attention。
  • direct pool-view decode CUDA Graph 默认开启。
  • SGLang-style piecewise prefill CUDA Graph。
  • SGLang-equivalent beam_results 输出。

Offline 性能

Qwen3-1.7B, ctx1000, beam256, batch4。

102.471 ms SGLang wall median

同 workload，SGLang beam-search PR 分支。

2.133x SGLang / GR

GR prefill 23.503 ms / decode 22.592 ms。

decode attention bucket：SGLang 34.864 ms vs GR 1.582 ms。两边都开 CUDA Graph， GR graph launches = 8、SGLang graph launches = 29；kernel launches GR 1261 / SGLang 1620。 差距主要来自 attention/KV 结构与 host launch path，而不是 graph API 本身。
